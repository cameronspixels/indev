CSS Grids

09/03/2017

by Cameron Barnes

The past couple of years have seen a great push in the implementation of CSS grids, with the technology having been around for a while now. Compatibility, understanding, and the desire for such a layout system on the web have all grown.

The power comes in the ability to so easily manipulate two dimensions. Where flexbox largely operates using rows and can represent a grid, grids works with both dimensions in a manner which removes the need to patch together other CSS properties. With progression like responsive design on the web currently, these simpler methods are welcome.

The Project

With this project, I wanted to make grids fluid. It’s great when design elements fit together perfectly with no gaps or flaws, and that’s what I was going for. This is all the more fun when responsive, because you should be able to see visual that everything changes as the viewport width scales.

I think of it like bricks in a house. If cements plays the role of the grid-gap, then it’s just all one grid. I note that brick-built houses aren’t (traditionally or easily) responsive… If you’ve used JavaScript libraries for a while, you’ll probably liken this to Masonry.js – one designed for grid layout.

To a large degree, this project is doing what Masonry aimed to enable. While I don’t use Masonry here (and haven’t previously), I wanted my demonstrator of grids to be photography portfolio similar to what Masonry would produce, but in CSS.

Grids in Design

Of course, the use of grids isn’t new. While the web hasn’t always been the greatest enabler, grids as a scheme of design have been around for decades upon decades.

Discussing the benefits of grids, Josef Muller-Brockmann notes the ability to control even space and organisation, allowing favourable placement of elements to better achieve objectives. That was published in 1981, and the same principles still apply. Grids were designed to cover all forms of visual media, and continue to today.

Getting Technical

When first learning about grids on the web, my go-to was ‘Get Ready for CSS Grid Layout’ by Rachel Andrew. As with most A Book Apart books, it’s affordable, concise, and fantastic when it comes to the topic at hand. This book in particular provides an overview of the simplest properties that grid requires to work, and is highly recommended if you’re yet to get off the ground.

Knowing a lot of that, to figure out the exact attributes that I needed for the fluidity that I was seeking, I spent a lot of time on Codepen. If you search for CSS Grid on there, I’ve probably looked at most of the first few pages to see how they used grids and the CSS properties defined. Particularly helpful from these was Stephanie’s CSS responsive image gallery, illustrating something similar to what I was looking to make.

My Demonstrator

As discussed, my aim was to use CSS grids to create a responsive photography portfolio. This was originally completed and can be found on Codepen (as below), but is also accessible at cameronspixels.co.uk/photography-grids/.

There were two main CSS properties that I’d highlight as enabling the responsive, flowing grid layout.

Grid-auto-flow determines the way in which children within the grid position themselves. Passing the keyword ‘row’ tells the grid to position children horizontally as they are defined, before moving onto another row.

‘Dense’ being passed instructs items to fill gaps as they appear in rows. Where larger items have to be pushed forward to fit into the grid, smaller items later in the HTML take the space that they leave.

Next came the use of ‘auto-fit' in the grid-template-colums property – that used to define the columns in the grid. The first value passed in the ‘repeat()’ argument is often a number, one which defines show many columns are to be created. The use of ‘auto-fit’ tells the grid to use however many can fit into the viewport.

grid-template-columns: repeat(auto-fit, minmax(100%, 1fr));

The ‘minmax()’ defines minimum and maximum size of the columns. Setting the minimum at 100% tell creates columns that are at least the size of the content inside, and a 1 fraction maximum ensures that all columns are the same size.

Combine the auto-fit and minmax arguments, and you get a grid of however many columns can fit into the viewport, where they (at least) fit the content within, and are also all of equal fractions.

Loading

Given that I was just importing images from Unsplash.com which (with quality in best interests) are generally around 2MB in size, I decided to add a loading animation to the gallery. Vanilla JavaScript defines that the animation shows and the grid hidden while the content is downloading – vice-versa when the downloading is complete.

The Product

It was exciting to see the grid layout take form. Having developed it throughout in CodePen, I could see changes work towards my end goal. On the whole, layout completes what I wanted it to do in terms of fluidity and responsiveness. Coming from just basic use of Grids and having never used libraries like Masonry, it was interesting to learn that just two properties altered the behaviour of the grid items and created the fluidity that I was looking for.

To me, this highlights the power of grids; a seemingly complex layout was in-fact created in such a small amount of code. To the length that some may have this perceived complexity as an issue, there’s a greater and greater argument for learning grid in what’s still early in it’s production life.

And there lies the only downside. After learning about grids, I wanted them everywhere. Of course, I had to stop at the very first project. A main requirement was that it had Internet Explorer compatibility as far back as IE 9 and 10, and this is one of few things pulling grids down. Only IE 11 has even partial grid support through the use of the -ms- prefix, as we can see below on the CanIUse (a great resource for finding out which browsers support web features). While around 87% compatibility is fantastic, alienating 13% can be a poor choice - so consider audience heavily. However with interest still only growing, grids remain a fantastic option and potentially the best looking towards CSS layout in the future.
